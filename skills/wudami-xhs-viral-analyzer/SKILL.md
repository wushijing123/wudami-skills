# 小红书爆款分析器（吴大咪增强版）

基于 CDP 抓取能力，融合吴大咪独创短视频方法论体系，从"找到爆款"到"输出可拍脚本"，分为**【板块1：原创生成】**和**【板块2：深度洗稿】**两大阶段一站式闭环。

**执行方式硬规则（深度洗稿板块）**：涉及 TikHub 的视频/笔记解析默认运行本地 Python 脚本直接请求 TikHub HTTP API，不走 MCP。除非主理人明确要求使用 MCP，否则不要调用 `mcp__tikhub*` 工具；MCP 链路通常更慢，且不复用脚本内置的下载、FFmpeg、ASR 与字段兼容逻辑。

## 核心能力

| 环节 | 能力 |
|------|------|
| 数据抓取 | 单标签页复用防弹窗、`playwright-stealth` 防风控、随机仿生滚动、深挖主页真实粉丝数 |
| 链接保真 | **优先提取带 `xsec_token` 的完整链接**，确保报告中每条笔记链接可直接打开，不会被扫码墙拦截 |
| 爆款筛选 | 自动提取真实粉丝数，精准执行 `< 5万` 过滤 |
| 脚本输出 | [板块1] 融合 17 种开头 + 5 种脚本模型 + 混合钩子 + 去 AI 味规范 |
| 深度洗稿 | [板块2] 跨端调用 TikHub+SiliconFlow，剥离原帖音轨ASR与封面文案，无缝套用《8种洗稿技巧》 |

## 工作流程（两步半自动模式）

> 为了对抗小红书极不稳定的 Web 端 UI 和 A/B 测试（如隐藏的"最新"按钮），本增强版**强制采用半自动接管模式**。

### 第一步：AI 启动浏览器 → 用户手动搜索

当用户提出分析需求时（如："抓取AI工具的低粉爆文"），AI **只执行打开浏览器**：

```bash
SKILL_DIR="/Users/wushijing/.claude/skills/wudami-xhs-viral-analyzer"
python3 "$SKILL_DIR/scripts/launch_chrome.py"
```

**AI 必须在此处暂停并提示用户：**
> "我已经为您打开了小红书网页。请您手动在搜索框输入关键词 `[用户的关键词]`，敲击回车，并在网页上点击筛选出 **最新**。
> 页面就绪后，请回复我「好了，开始抓取」。"

⚠️ **绝对不要尝试自动输入关键词或自动点击"最新"按钮**——小红书的 UI 结构不稳定，自动化操作极易失败。

### 第二步：用户确认后 → AI 自动抓取

收到用户确认指令后，AI **省去 keyword 参数**，无缝接管当前搜索页面：

```bash
SKILL_DIR="/Users/wushijing/.claude/skills/wudami-xhs-viral-analyzer"
cd "$SKILL_DIR/scripts"

python "$SKILL_DIR/scripts/xhs_scraper.py" \
  --cdp http://localhost:9333 \
  --mode search \
  --auto-scroll \
  --target-count 100 \
  --deep-fetch \
  --output /tmp/xhs_search_result.json
```

> 🛡️ **防风控提示**：脚本已内置隐身衣、幽灵鼠标、仿生随机滚动和长停顿，全程在当前标签页静默执行，不会弹窗打断操作。

### 第三步：生成增强版分析报告

先运行分析脚本提取数据：
```bash
python "$SKILL_DIR/scripts/xhs_analyzer.py" \
  -i /tmp/xhs_search_result.json \
  --keyword "用户搜索的关键词" \
  --max-fans 50000 \
  --min-likes 500
```

然后 **AI 必须将报告写入用户的 Obsidian 系统**。

## 📊 报告结构（必须按此顺序）

1. **📊 低粉爆款榜单**（开篇第一部分，必须有！）
   - 从 JSON 数据中筛选 `fansCount < 5万` 且 `likeCount > 500` 的笔记
   - 按点赞数降序排列
   - 表格包含：序号、标题、点赞数、博主、粉丝数、链接
   - **链接必须是带 `xsec_token` 的完整 URL**，不能是裸 `/explore/笔记ID` 链接

2. **🔥 数据洞察速览**（简要总结超级黑马、爆文类型、高频标题词）

3. **🎯 三个选题方向**（每个方向含开头公式、脚本模型、混合钩子、口播框架）
   - 必须引用榜单中的真实笔记数据作为"弹药"（标注序号、粉丝数、点赞数）

4. **📚 附录：方法论速查**（17种开头 + 5种脚本模型 + 去AI味清单）

> ⚠️ **硬规则**：报告中如果没有低粉爆款榜单表格，就是不合格的报告。榜单是核心资产，选题方向是附加分析。

> ⚠️ **链接硬规则**：报告中所有笔记链接必须包含 `xsec_token` 参数。裸链接会被小红书扫码墙拦截，属于不合格报告。

## 🔁 第五步：进入【板块2：洗稿板块】

当用户确认进入洗稿板块，并提供了笔记序号（或链接）时，AI 需按以下纯自动工作流执行：

### 1. 提取原文素材库 (Deep Extract)
AI 根据用户提供的序号，从上文生成的表格中提取出带 `xsec_token` 的链接或笔记 ID，并在后台执行内置的独立萃取脚本：
```bash
SKILL_DIR="/Users/wushijing/.claude/skills/wudami-xhs-viral-analyzer"
python3 "$SKILL_DIR/scripts/single_note_extractor.py" "这里填提取到的链接或笔记ID"
```
*(该脚本内置了全套独立工业管线，通过 TikHub + SiliconFlow，自动剥离音视频并返回 JSON 格式的原帖标题、正文、提取口播 ASR 以及封面视觉文案。不需要依赖任何外部本地 Skill。)*

### 2. 读取洗稿方法论
AI 必须在后台 `view_file` 读取本 Skill 目录下的洗稿知识库：
`references/8种洗稿技巧.md`

### 3. 直接对话生成洗稿内容
无需再调用外部大模型，AI 直接将提取到的 `JSON 萃取数据` + `8种洗稿技巧` 结合起来，**代入当前上下文**，向用户确认选用哪种洗稿技巧，或直接给出一版采用高维技巧（如视角转换/知识迁移）的全文。

**【强制输出格式】**：
生成的文件必须严格采用以下**“1:1 复刻对照排版”**结构：

1. **`## 📥 【原版素材】(供1:1复刻参考)`**：展示提取到的“封面文案”和真实的“原视频语音(ASR)”（如果有的话）。
2. **`## 🛠 【洗稿方案】`**：清晰标注使用了8种洗稿方法论中的哪一种，并说明改写思路。
3. **`## 📝 【洗稿后口播稿】 (推荐使用)`**：严格符合大咪口语风格、去AI味的最终成品。
4. **去AI味自检清单**：必须附带检查清单。

- **保存成文件**：`01-待深化的选题/低粉爆文搜集分析/YYYY-MM-DD-小红书-搜索词/YYYY-MM-DD-小红书-搜索词-洗稿-原标题.md`

## 常见问题

### 链接打开显示扫码墙或跳转博主主页（xsec_token DOM 提取陷阱）
- **核心避坑点**：小红书搜索页的笔记卡片 `DOM` 中通常包含**两个**带有 `xsec_token` 的链接：一个是真正的笔记链接（通常格式为 `/search_result/ID`），另一个是博主主页链接（`/user/profile/ID`）。
- **提取逻辑硬要求**：JS 提取时**绝对不能**匹配 `/user/profile/`，否则会错误抓取到作者主页导致无法查看原文。正确的做法是匹配带有 `xsec_token` 且不包含 `/user/profile/` 的链接。
- **URL 转换**：提取到 `/search_result/` 链接后，需统一在代码中执行 `.replace('/search_result/', '/explore/')`，将其转换为可全平台访问的标准笔记链接。
- **字符截断陷阱**：`xsec_token` 非常长（约 44 字符），提取 `rawHtml` 留存时，`outerHTML.substring` **至少必须提取前 2000 个字符**。如果提取过短，token 会被截断（如丢失末尾的 `=` 或字母），导致链接彻底报废，且由于 token 是动态生成的，截断后无法事后补回。

### 抓取结果为空
1. 确认用户已手动搜索并切换到"最新"
2. 检查页面是否在搜索结果页
3. 浏览器控制台：`document.querySelectorAll('section.note-item').length`

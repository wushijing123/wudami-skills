# `wudami-lark-single-video-api` 极客单兵管道

这是吴大咪体系中用于“异质化媒介提取”的绝对尖刀工具。

## 🎯 最佳触发条件
当主理人使用手机背部截获或者定时巡检了一批数据，并对大模型说：“把刚才传进新库里那个空字段的口播给我抓满”，或者你在维护飞书表格、并且碰见一个孤立小红书视频链接时。

## 🚀 调用指令
直接在当前会话的底层命令行中召唤单兵脚本：

```bash
python3 ~/.claude/skills/wudami-lark-single-video-api/scripts/single_video_asr.py --url "https://www.xiaohongshu.com/explore/xxxxxxxx"
```

## ⚠️ 大模型机器互联特规（必读！！！）
1. 本脚本具备着极其变态的**强阻断机制**。一旦运行环境或者底层没有正确导入 `TIKHUB_API_KEY` 或 `SILICONFLOW_API_KEY`，该脚本会无情地瞬间掐断所有流媒体并抛出中断报错。
2. **它的极净体质**：无论这个视频转写经历了切分、调用、转化，这个指令**绝不会**用乱七八糟的进度条或文字刷屏。
3. 如果执行提取且目标是一个真实带有口播的短视频，它的 Stdout（标准输出）只会吐出一长串没有任何回车的：`🎼在广州这个...`
4. 请不要让主理人去阅读输出，你需要将脚本打印出来的这串 `🎼...` 的珍贵结果，**利用 `lark-base` 技能稳当地写进飞书多维表格（Base表）的对应单元格中。**

## 🛡️ 这个工具替你抗下的坑
你不用去写什么冗杂的代理逻辑。脚本默认使用 TikHub 大陆域名 `https://api.tikhub.dev`，并按 `App V2 → App → Web V3 → Web V4 → HTML MP4` 的顺序动态解析视频流；脚本会在内部使用 FFmpeg 的无损脱壳流将数据压榨后直接传给 `SiliconFlow/FunAudioLLM`，省下了极大的开销。主机无需下载这百兆的巨型 MP4 文件就能提取声音！

**执行方式硬规则**：默认运行本地 Python 脚本直接请求 TikHub HTTP API，不走 MCP。除非主理人明确要求使用 MCP，否则不要调用 `mcp__tikhub*` 工具。

# 小红书单篇笔记深度拆解（TikHub API 付费版）

**执行方式硬规则**：默认运行本地 Python 脚本直接请求 TikHub HTTP API，不走 MCP。除非主理人明确要求使用 MCP，否则不要调用 `mcp__tikhub*` 工具；MCP 链路通常更慢，且不复用脚本内置的降级链、缓存和媒体处理逻辑。

**TikHub 配置**：默认使用 `https://api.tikhub.dev`，可用 `TIKHUB_BASE_URL` 覆盖；默认超时 `45s`，可用 `TIKHUB_TIMEOUT` 覆盖。视频 URL 解析兼容 `video_info_v2`、`video`、`video_info`、`videoInfo`、`noteCard.video`。

## 核心工作流

```
用户提供笔记URL → 清缓存 → TikHub API 抓取笔记详情+评论 → 视觉拼接 → AI 8段式深度拆解 → 桌面导出 .md + .docx + .html
```

## 与 CDP 版的区别

| 维度 | CDP 版（免费） | API 版（本 Skill） |
|------|---------------|-------------------|
| 浏览器 | 需要 Chrome | **不需要** |
| 登录 | 首次需要扫码 | **不需要** |
| 验证码 | 可能遇到滑块 | **不会遇到** |
| 费用 | 免费 | ~$0.002/次 |
| 速度 | ~15 秒 | **~3 秒** |

## 🚫 前置死规则：API Key 检查（不通过则一切停摆）

> ⛔ **这是整条流水线的第一道门禁，也是绝对不可绕过的死规则，比提取 URL 还早。没有配置 TIKHUB_API_KEY，后续所有步骤（包括清缓存、抓数据、写报告）无论如何都全部禁止执行！第一关过不去，决不进行其他操作！**

在做任何事之前，**必须先执行以下命令**检查环境变量是否存在：

```bash
echo "TIKHUB_API_KEY=${TIKHUB_API_KEY:+已配置}" && echo "SILICONFLOW_API_KEY=${SILICONFLOW_API_KEY:+已配置}"
```

**判定规则：**

| 检查结果 | AI 必须做的事 |
|----------|-------------|
| `TIKHUB_API_KEY=已配置` | ✅ 通过，继续第零步 |
| `TIKHUB_API_KEY=`（空） | ❌ **立即停止一切操作**，向用户输出以下完整引导文案后结束对话，不执行任何后续步骤 |

**如果 TIKHUB_API_KEY 为空，必须输出：**

```
❌ 无法执行笔记拆解：未检测到 TIKHUB_API_KEY 环境变量！

本 Skill 依赖 TikHub 付费 API 进行数据采集（每次约 $0.002），必须先配置 API Key 才能运行。

🔧 配置方法（一次性操作，以后永久生效）：
1. 注册 TikHub 账号：https://user.tikhub.io/
2. 在控制台获取你的 API Key
3. 在终端执行：
   echo 'export TIKHUB_API_KEY="你的Key"' >> ~/.zshrc && source ~/.zshrc
4. 配置完成后，重新告诉我要拆解的笔记链接即可！

💡 如果你不想付费，可以使用免费的 CDP 浏览器版：wudami-xhs-note-analyzer-cdp（需要登录小红书）
```

> ⚠️ **SILICONFLOW_API_KEY** 不是硬性阻断项（图文笔记不需要），但如果用户要拆解**视频笔记**且该变量为空，脚本会在转录环节以退出码 3 强制中断并提示配置。

## 🔴 第零步：提取用户 URL（执行任何命令前必须完成）

1. 找到用户在对话中提供的小红书链接。
2. **提取纯 URL**：如果是富文本链接，只提取 `https://...` 部分。
3. 对于 API 版，脚本会**自动从 URL 中提取 `xsec_token`**（如果存在）。
   - 有 `xsec_token` 时：先尝试 app 端点，失败后自动降级到 web_v4 端点（覆盖率更高）
   - 没有 `xsec_token` 时：仅使用 app 端点（大部分笔记够用）
   - **建议：尽量提供带 `xsec_token` 的完整链接**，这样容错率最高

## 第三步：API 抓取笔记数据

```bash
python3 scripts/xhs_note_api_scraper.py \
  --url "用户提供的完整笔记URL" \
  --max-comments 20 \
  --output outputs/note_data.json
```

**抓取内容**：标题、正文全文、话题标签、互动数据（赞/藏/评/转）、作者信息、封面+内页图列表、视频链接、评论区 TOP 评论。

脚本会自动完成以下附加工作：
- **图文笔记**：自动下载所有配图 → Pillow 拼接为 `storyboard.jpg`
- **视频笔记**：自动下载视频 → ffmpeg 截帧拼接故事版 + 音频提取 → 自动转录口播文案

## 第四步：AI 深度分析（8段式拆解框架）

**⚠️ 极度重要：保存分析结果**
请你（AI）在完成分析后，**必须第一步将生成的完整的 8 段式 Markdown 报告保存到文件 `outputs/笔记拆解报告.md` 中！** 绝不能写到 `/tmp` 或其他路径。

读取 `outputs/note_data.json`，写出**8段式爆款笔记深度拆解报告**。

### 必须包含以下内容：

## 核心数据总览（必须使用 Markdown 表格样式）

| 数据维度 | 指标详情 |
| :--- | :--- |
| **笔记标题** | [提取标题] |
| **博主昵称** | [提取博主] |
| **发布时间** | [提取时间] |
| **笔记形态** | [提取图文/视频] |
| **互动指标** | 赞[数量]、收藏[数量]、评论[数量]、分享[数量] |

## 8段深度分析：

#### 一、选题与痛点洞察
- 选题方向归类：痛点型 / 热点型 / 人性型 / 干货型
- 精准击中的用户需求是什么？
- 与同类选题相比的差异化创新点
- **爆款公式提炼**：`______赛道 + ______痛点 + ______形式`

#### 二、标题钩子拆解
- 原标题展示
- **提炼标题结构**（如：数字+痛点+利益承诺）
- **提炼标题模板**（可直接套用的公式）
- 关键词布局分析：人群词 / 场景词 / 痛点词 / 搜索词
- 情绪价值：好奇 / 焦虑缓解 / 获得感 / 身份认同

#### 三、视觉推演逐段解析

> ⚠️ **动态输出规则**：请先读取 `note_data.json` 中的 `noteType`。根据其是【视频】还是【图文】，分别使用以下两套**完全不同**的表格表头与视觉分析逻辑，绝不要混用！
>
> **情况A：如果是「视频笔记」**：
| 时间段 / 序号 | 镜头类型 | 画面描述 |
| :--- | :--- | :--- |
| (如0:00-0:03) | (如实景口播/屏幕录制/B-roll空镜) | (客观描述画面内容) |
>
> **情况B：如果是「图文笔记」**：
| 页面序号 | 页面类型 | 视觉排版拆解 |
| :--- | :--- | :--- |
| 封面 | 🎯 流量点击层 | 封面黄金三要素拆解 |
| 图1 | 📖 信任构建层 | 排版与信息分层分析 |
| 尾图 | 🛒 转化行动层 | 互动引导与引流分析 |

#### 四、正文逻辑与文案拆解
> 根据笔记类型分别关注不同的拆解重点（视频关注口播节奏，图文关注翻页钩子）

#### 五、评论区与用户心理
- TOP热门评论摘录（3-5条）
- 评论区氛围与互动诱饵分析

#### 六、算法与运营策略
- 话题标签选择分析
- 发布时间与助推手段

#### 七、商业价值拆解
- 合作笔记预警
- 变现路径判断

#### 八、可复刻方法论与避坑
- 可直接迁移的元素清单
- 优化建议与风险预警
- **一句话总结**：`核心爆点 = ______，可复刻动作 = ______，需规避风险 = ______`

## 异常处理

| 情况 | 退出码 | 处理 |
|------|--------|------|
| TIKHUB_API_KEY 未配置 | 1 | 引导用户配置环境变量 |
| API 返回 401 | — | API Key 过期或余额不足，引导充值 |
| API 返回 400 | — | 笔记已删除或不存在 |
| 没有转录API | 3 | 引导用户配置 SILICONFLOW_API_KEY |

## 防幻觉约束

1. **正文必须基于实际抓取**：如果正文抓取为空，必须标注「正文未成功抓取」
2. **评论必须基于实际数据**：禁止编造评论内容
3. **视频口播**：未提取到口播文案时，必须标注「基于标题+封面推测」
4. **互动数据**：必须使用实际抓取的数字，禁止估算
5. **缺少转录 API 的免责声明**：如果这是一条「视频笔记」，且因用户没有配置 `SILICONFLOW_API_KEY` 而未能提取到真实口播片段，那么在生成的 8 段深度拆解报告中，**必须**在醒目位置（如开头或结尾）单独加粗一段提醒用户：
   「⚠️ **友情提示：由于系统未检测到你的硅基流动 API (SILICONFLOW_API_KEY)，本次分析被迫跳过了对视频原声口播的提取。这导致本报告对口播文案、情绪等维度的拆解可能不够深入。建议你在大咪系统中配置 API 以解锁“完全体”的深拆分析！**」

# 小红书单篇笔记深度拆解（CDP免费版）

## 核心工作流

```
用户提供笔记URL → 启动独立Chrome(9333) → 登录检测 → 抓取笔记详情+评论 → （视频笔记）ffmpeg提取音频 → AI 8段式深度拆解 → 桌面导出 .md + .docx
```

---

## ⚠️ 踩坑防护清单（必读）

| # | 规则 | 违反后果 |
|---|------|----------|
| 1 | Chrome 用独立端口 **9333**，禁用 9222 | 杀死用户默认浏览器 |
| 2 | requests 必须 `trust_env=False` 绕过系统代理 | CDP 连不上 |
| 3 | 抓取前检测登录状态（退出码 2 = 未登录） | 崩溃无提示 |
| 4 | 视频笔记必须用 ffmpeg 提取音频再转文字 | 丢失口播内容 |
| 5 | 评论区需等待加载后再抓取 | 评论为空 |
| **6** | **`--url` 参数必须把用户给的链接原封不动、一字不差地传入，禁止自己截断、清洗、重组 URL** | xsec_token 失效，触发风控 |

## 🔴 第零步：提取用户 URL（执行任何命令前必须完成）

**在运行任何脚本之前，必须先完成这一步：**

1. 找到用户在对话中提供的小红书链接。
2. **严禁自己发挥或精简 URL**：**原样复制这整条链接**，必须包含 `?` 后面所有的查询参数（特别是 `xsec_token`）。
3. ⚠️ 如果用户提供的是带有标题的富文本链接或 Markdown 链接形式（如 `[🔗 学了书桌拍摄大法...](https://www....)`），必须**只提取括号中的纯 `https://...` URL** 作为参数传入，绝不能把括号外的文本传进去！
4. 用这条**完整链接**替换下方命令中的 `--url` 参数值。

> ❌ **错误示例（禁止出现，会导致拦截）**：
> ```bash
> --url "https://www.xiaohongshu.com/explore/69a98363000000001a029638"
> ```
> ✅ **正确示例（必须包含长参数）**：
> ```bash
> --url "https://www.xiaohongshu.com/explore/笔记ID?xsec_token=完整xsec_token&xsec_source=pc_user"
> ```

## 🧹 第一步：清空历史缓存（极其重要，防止数据张冠李戴）

⚠️ **强制规定**：因为 OpenClaw 如果之前执行过其他笔记的拆解，`/tmp` 或 `outputs/` 目录中会残留上一个笔记的 `note_data.json` 和 `笔记拆解报告.md`。如果不删除，最后的桌面导出脚本会把**前一家的数据张冠李戴到这一家头上（导致出现"账号都是错误的"问题）**。

因此，在每次处理用户的全新任务时，**你必须首先执行以下清除命令：**

```bash
rm -rf outputs/note_data.json outputs/笔记拆解报告.md outputs/storyboard.jpg outputs/_frames /tmp/note_data.json /tmp/笔记拆解报告.md
```

## 第二步：环境准备（全平台真正的开箱即用）

> **💡 OOTB (Out of the box) 特性**：
> Python 脚本已实现**全自动依赖安装**（Auto-bootstrap）。由于引入了 `imageio-ffmpeg`，即使是 Windows 或没装 Homebrew 的 Mac，也会**自动下载免安装版的 ffmpeg 二进制文件**，不再需要修改系统 PATH。
> 用户（包括 OpenClaw 用户）**只需要电脑上有 Python3** 即可，脚本首次运行时会自动执行 `pip install` 和浏览器环境初始化。

*无需任何手动安装，直接开始第三步即可。*

## 第三步：启动独立 Chrome

```bash
python3 scripts/launch_chrome.py
```

- 独立端口 9333 + 独立 profile `~/.xhs-chrome-profile/`
- 与 wudami-xhs-analyzer-claw 共享同一个浏览器实例和登录态
- 已在跑 → 秒跳过；未启动 → 自动启动并等待就绪

## 第四步：抓取笔记数据

> ⚠️ **URL 传入铁律（违反必崩）**：`--url` 参数必须传入用户提供的**完整原始 URL**，包含所有查询参数（`xsec_token`、`xsec_source`、`source` 等）。
> **绝对禁止**只传 `explore/笔记ID` 这种简化版本！小红书服务端依赖 `xsec_token` 做鉴权，缺少参数会直接触发风控验证码拦截。

```bash
python3 scripts/xhs_note_scraper.py \
  --cdp http://localhost:9333 \
  --url "https://www.xiaohongshu.com/explore/笔记ID?xsec_token=完整xsec_token&xsec_source=pc_search&source=web_search_result_notes" \
  --max-comments 20 \
  --output outputs/note_data.json
```

**抓取内容**：标题、正文全文、话题标签、互动数据（赞/藏/评/转）、作者信息、封面+内页图列表、视频链接、评论区TOP评论。

## 第五步：视频笔记处理（全自动，仅视频笔记触发）

脚本检测到 `note.isVideo == true` 时，会自动执行以下完整管线：

### 4a. 音频提取 + 口播文案转录
```
视频下载 → ffmpeg 提取 WAV 音频 → 4级降级自动转录：
  1️⃣ 硅基流动 SenseVoice（免费，推荐）
  2️⃣ OpenAI Whisper API（通用备选）
  3️⃣ 本地 whisper CLI（完全离线）
  4️⃣ 优雅跳过（不影响流程）
```

### 4b. 视觉素材自动整合（Storyboard / 拼接卡）
无论是视频笔记还是图文笔记，脚本都会**自动将繁杂的视觉素材整合为一张大图（storyboard.jpg）**，极大降低给大模型喂图的 Token 消耗：
- **视频笔记**：纯 ffmpeg 每3秒截取1帧 → 自动拼接成九宫格大视频缩略图。
- **图文笔记**：自动批量下载所有内置配图 → 通过 Pillow 裁剪并排版缩放 → 拼接成多行多列的图文网格矩阵卡片。

这使得大模型能像人一样，一眼“扫”完整个视频时间轴或整套左滑图文。

### 4c. 视觉分析（由大模型完成）

大模型收到 `storyboard.jpg` 后，参考 `prompts/视觉分析提示词.md` 进行 6 维度分析：
1. 镜头语言逐段解析
2. 剪辑节奏与转场
3. 文字与视觉元素
4. 色调与视觉风格
5. 开头3秒与结尾设计
6. 可复刻视觉清单

分析结果写入报告的「四、视觉呈现分析」段落。

## 第六步：AI 深度分析（8段式拆解框架）

**⚠️ 极度重要：保存分析结果**
请你（AI）在完成分析后，**必须第一步将生成的完整的 8 段式 Markdown 报告保存到文件 `outputs/笔记拆解报告.md` 中！** 绝不能写到 `/tmp` 或其他路径。只有保存之后，才能执行下一步桌面打包交付。

读取 `note_data.json`，写出**8段式爆款笔记深度拆解报告**。

> **框架设计说明**：此框架融合了4套经过验证的拆解提示词精华，覆盖选题→标题→正文→视觉→互动→算法→商业→复刻的完整链路。

### 必须包含以下内容：

## 核心数据总览（必须使用 Markdown 表格样式）

| 数据维度 | 指标详情 |
| :--- | :--- |
| **笔记标题** | [提取标题] |
| **博主昵称** | [提取博主] |
| **发布时间** | [提取时间] |
| **笔记形态** | [提取图文/视频] |
| **互动指标** | 赞[数量]、收藏[数量]、评论[数量]、分享[数量] |

## 8段深度分析：

#### 一、选题与痛点洞察
- 选题方向归类：痛点型 / 热点型 / 人性型 / 干货型
- 精准击中的用户需求是什么？
- 与同类选题相比的差异化创新点
- **爆款公式提炼**：`______赛道 + ______痛点 + ______形式`

#### 二、标题钩子拆解
- 原标题展示
- **提炼标题结构**（如：数字+痛点+利益承诺）
- **提炼标题模板**（可直接套用的公式）
- 关键词布局分析：人群词 / 场景词 / 痛点词 / 搜索词
- 情绪价值：好奇 / 焦虑缓解 / 获得感 / 身份认同

#### 三、视觉推演逐段解析

> ⚠️ **动态输出规则**：请先读取 `note_data.json` 中的 `noteType`。根据其是【视频】还是【图文】，分别使用以下两套**完全不同**的表格表头与视觉分析逻辑，绝不要混用！
>
> **情况A：如果是「视频笔记」**，请关注剪辑、景别与时间流，严格使用以下表格输出：
| 时间段 / 序号 | 镜头类型 | 画面描述 |
| :--- | :--- | :--- |
| (如0:00-0:03) | (如实景口播/屏幕录制/B-roll空镜) | (客观描述当前时间的画面内容，以及花字特效) |
>
> **情况B：如果是「图文笔记」**，由于没有时间线，核心在于排版、信息流顺序与重点提取，请严格使用以下表格输出：
| 页面序号 | 页面类型 | 视觉排版拆解 |
| :--- | :--- | :--- |
| 封面 | 🎯 流量点击层 / 痛点放大 / 场景代入 | 封面黄金三要素拆解（主背景+大字号标题+附号利益点）、配色视觉冲击力... |
| 图1 | 📖 信任构建层 / 过程展示 / 原理科普 | 花字引导线设计、是否采用了前后对比排版、信息分层是否密集... |
| 尾图 | 🛒 转化行动层 / 评论引导 | 强引导互动符号、引流私域暗示钩子... |

#### 四、正文逻辑与文案拆解

> 同样根据笔记类型，分别关注不同的拆解重点：
>
> **如果为视频笔记**（结合语音转录的口播文案）：
> - **开头技巧**：前3秒抛出了什么钩子留人？（提问/反常识/痛点场景）
> - **口播节奏**：信息密度如何？有没有碎嘴感、语气词？
> - **人设信息**：口播中植入了哪些身份标签和信任背书
> 
> **如果为图文笔记**（结合画面中的文字排版）：
> - **翻页钩子**：左滑欲望是怎么被勾起来的？（例如制造了悬念或者连载指示）
> - **内页信息密度**：文字在一张大图里是“填鸭罗列”还是“留白呼吸感”？
> - **结尾设计**：尾图或文案是否直接促成了赞藏评论的转化行动？

#### 五、评论区与用户心理
- **TOP热门评论摘录**（3-5条）
- 评论区整体氛围：需求型多还是看戏型多
- 是否设置了「低成本互动诱饵」（评论抽奖/猜价格/求链接）
- 是否存在争议性讨论引发站队
- **情感共鸣点**：内容引发了什么圈层认同（打工人/学生党/宝妈等）
- 提供的价值组合：实用型 vs 情绪型 vs 娱乐型的比例

#### 六、算法与运营策略
- 话题标签选择：是否兼顾「大流量词 + 精准长尾词」
- 发布时间是否踩中活跃时段
- 是否通过合集/连载/评论区置顶助推流量
- 是否结合热点事件或节日营销

#### 七、商业价值拆解
- **合作笔记预警（重要）**：检查笔记话题标签或内容，如果涉及到具体品牌（如 `#海尔`、`#豆包` 等带 `#` 符号的品牌名），大概率是蒲公英合作笔记。请在这一段开头醒目地使用提醒块，例如：`> ⚠️ **合作笔记预警**：检测到涉及品牌话题 #[品牌名]，该笔记大概率为蒲公英商业笔记。`
- 广告植入是否自然（自用分享/无广声明/3步软化法）
- 私域引流路径（是否隐蔽）
- 单篇笔记对粉丝增长的拉动效果
- 变现路径判断：广告/电商/私域/知识付费

#### 八、可复刻方法论与避坑
- **可直接迁移的元素清单**（标题模板/封面设计/正文框架/开头结尾公式）
- **如果让你做一篇类似笔记的优化建议**（从标题、正文框架、开头结尾、人设引流上具体给出落地建议）
- **风险预警**：内容是否存在隐性违规（虚假对比/绝对化用词/过度广告）
- **一句话总结**：`核心爆点 = ______，可复刻动作 = ______，需规避风险 = ______`

## 异常处理

| 情况 | 退出码 | 处理 |
|------|--------|------|
| Chrome 未启动 | 1 | 执行 `python3 scripts/launch_chrome.py` |
| 未登录或验证码拦截 | **0 (挂起)** | 脚本遇到未登录或`xsec`滑动验证码时，**不会退出报错**，而是**在后台无限挂起等待**。此时浏览器会保持原样不动，用户可以直接在浏览器手动扫码/滑动滑块。验证通过后，工具会自动汲取数据并继续执行。 |
| 笔记已删除 | — | 页面无内容，告知用户 |
| 视频下载失败 | — | 标注未提取，基于标题+封面推测分析 |
| 没有转录API (API_KEY缺失) | — | **降级容错**：脚本将跳过转录继续生成视觉大纲资产，大模型必须在最终分析报告中明确提示用户配置 `SILICONFLOW_API_KEY` 环境变量解锁视频深度解析。 |
| ffmpeg 不可用 | — | 跳过音频提取，标注分析局限 |

## 防幻觉约束

1. **正文必须基于实际抓取**：如果正文抓取为空，必须标注「正文未成功抓取」
2. **评论必须基于实际数据**：禁止编造评论内容
3. **视频口播**：未提取到口播文案时，必须标注「基于标题+封面推测」
4. **互动数据**：必须使用实际抓取的数字，禁止估算
5. **浏览器隔离**：端口 9333 + 独立 profile，绝不碰默认 Chrome
6. **缺少转录 API 的免责声明**：如果这是一条「视频笔记」，且因用户没有配置 `SILICONFLOW_API_KEY` 而未能提取到真实口播片段，那么在生成的 8 段深度拆解报告中，**必须**在醒目位置（如开头或结尾）单独加粗一段提醒用户：
   「⚠️ **友情提示：由于系统未检测到你的硅基流动 API (SILICONFLOW_API_KEY)，本次分析被迫跳过了对视频原声口播的提取。这导致本报告对口播文案、情绪等维度的拆解可能不够深入。建议你在大咪系统中配置 API 以解锁“完全体”的深拆分析！**」

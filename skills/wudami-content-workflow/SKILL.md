# 吴大咪内容工作流

大咪一人公司内容生产流水线，覆盖从选题到发布的完整链路。

---

## Step 1：找选题

**触发词**：「找选题」「帮我找选题」「推荐选题」「深化选题」

### 选题来源（全部实时抓取，不读本地选题库）

#### ① 小红书热榜
优先用本地 Python/HTTP 方式直连 TikHub API（默认 `https://api.tikhub.dev`，环境变量 `TIKHUB_BASE_URL` 可覆盖），不要默认调用 `mcp__tikhub*`。MCP 仅在主理人明确要求时使用。

返回实时热榜 20 条。适合：发现平台正在推什么。

#### ② 全网热榜（并行抓取）
```
小红书热榜：本地 Python/HTTP 直连 TikHub，MCP 仅显式要求时使用
虎嗅：WebFetch https://www.huxiu.com
AIbase：WebFetch https://www.aibase.com/news
机器之心：WebFetch https://www.jiqizhixin.com
```

#### ③ AI工具资讯（并行抓取）
```
AIbase：WebFetch https://www.aibase.com/news
机器之心：WebFetch https://www.jiqizhixin.com
虎嗅：WebFetch https://www.huxiu.com
```

### 已验证失败的来源

| 来源 | 失败原因 | 处理方式 |
|------|---------|---------|
| 量子位 | 服务过载（529） | 可跳过，等下次再试 |
| 其他未列出平台 | 需登录/JS渲染 | **立即提示用户登录**，登录后在浏览器打开确认 |

**规则：遇到需要登录的来源，立刻停下来告诉用户"需要先登录"，不要反复重试、不要跳过，要用户确认登录后再继续。**

### 选题筛选标准

筛选后的选题必须满足：
1. **大咪能讲**：跟她的背景（电商6年/新媒体4年/28天训练营）有结合点
2. **平台匹配**：适合小红书 或 公众号，不能通用模糊
3. **受众明确**：针对某一类具体人群，不是"大家"
4. **有角度**：不是泛话题，能找到独特切入角度

### 输出格式

每个推荐选题：
```
## 选题 {N}
- 标题：[选题名称]
- 来源：[来自哪个热榜/平台] → [链接]
- 角度：[切入角度，一句话]
- 目标人群：[谁会看]
- 平台分流：[小红书 / 公众号]
- 核心观点：[要传达什么]
- 预期效果：[传播效果预估]
```

来源链接规则：
- 小红书热榜：`https://www.xiaohongshu.com/explore/` + note_id
- 微博热搜：`https://s.weibo.com/weibo?q=` + URL编码话题
- B站热榜：从 `<a href="//www.bilibili.com/...">` 提取完整链接
- 知乎热榜：从页面元素提取真实链接
- 36氪：从页面提取文章链接
- AIbase：从页面提取资讯原文链接
- 虎嗅/机器之心：从页面提取文章链接

每次推荐 **3-5 个选题**，附上来源链接供你验证。

## Step 2：基于选题写稿子

**触发词**：「深化选题」「帮我写稿子」「写脚本」「生成脚本」

### 执行前必读（4个文件必须检索）

1. `02-素材库/金句库/大咪口播风格规范.md` — 口播风格最高优先级
2. `02-素材库/核心概念库/账号安全红线清单.md` — 违规词必查
3. `02-素材库/核心概念库/核心概念库.md` — 品牌关键词
4. `04-业务运营/关于我.md` — 信任背书数据（真实年限/背景/学员数）

### 输出位置

- 小红书 → `01-内容生产/01-待深化的选题/YYYY-MM-DD-小红书-选题名.md`
- 公众号 → `公众号内容/01-待深化选题/YYYY-MM-DD-公众号-选题名.md`

### 脚本结构

```markdown
## 元信息
- 状态：待深化 / 制作中 / 已发布
- 平台：小红书 / 公众号
- 脚本类型：痛点解决型 / 认知升级 / 清单型 / 故事型
- 目标用户：
- 发布日期：

## 一句话诱因（开头）

## 分论点（每个200-400字完整展开）

## 结尾
有需要看号或者不懂怎么选赛道做定位的。有问题问大咪，有问必答哦。
```

## Step 3：写开头钩子

**触发词**：「写开头」「帮我写开头」「优化开头」

### 执行依据

读取 `02-素材库/00-提示词模板/7-短视频/01-17种开头方法论.md`

### 输出格式

输出17种开头方案：
```
【开头类型】核心方法名
{完整黄金开头文案，约40-60字，口播15秒内}
```

### 去AI味要求

- 绝对禁止书面语（"由此可见"、"总之"）
- 必须完全口语化，像聊天
- 情绪调动，用词精准有力

---

## Step 4：写标题

**触发词**：「写标题」「帮我写标题」「生成标题」

### 执行依据

读取 `02-素材库/00-提示词模板/5-标题/01-标题方法论版.md`（20种标题方法论）

### 输出格式

```
1：标题方法：[用的哪种方法论]
创作标题：[标题，不超过20字]
创作思路：[为什么用这个方法]
```

### 要求

- 不超过20字
- 不用AI常见套路词（"秘诀"、"秘籍"、"干货"）
- 针对某一类人群表达
- 有参与感

---

## Step 5：写正文

**触发词**：「写正文」「写小红书正文」「生成正文」

### 小红书正文规范

根据爆款脚本库真实结构：

- **开头**：一句话诱因（完整段落，1-2段）
- **正文**：3-10个分论点，每个分论点200-400字完整展开
- **结尾**：核心观点一句话 + 固定口癖

### 脚本字数参考

| 类型 | 字数 |
|------|------|
| 90秒视频 | 约400-450字 |
| 60秒视频 | 约280-320字 |
| 图文笔记 | 正文200-400字 |

### ⚠️ 硬规则

- **禁止** `---` 分隔符（渲染成HTML横线）用空行代替
- 不用序号列表
- 不用感叹号连用

---

## Step 6：渲染生成文章

### 6a：公众号文章

**触发词**：「发公众号文章」「渲染公众号」

使用链路：
1. `baoyu-article-illustrator` — 配图（3张，16:9，notion风格）
2. `baoyu-cover-image --aspect 2.35:1` — **封面必须用2.35:1**
3. `baoyu-markdown-to-html --theme default --title "标题"`
4. `baoyu-post-to-wechat` — 发草稿箱

**⚠️ 硬规则**：
- `article.md` 禁止 `---` 分隔符
- 必须有 YAML frontmatter `title`
- HTML转写用 `--title` 参数传入正确标题（不用临时"temp-article"）

### 6b：小红书

**触发词**：「生成小红书」「渲染小红书」「生成短文」

生成 Markdown，格式：
```markdown
# [标题]

[正文内容]

#话题1 #话题2
```

## Step 7：生成白板（Excalidraw）

**触发词**：「生成白板」「生成excalidraw」「生成脑图」「生成可视化」

### 执行方式

生成 `.excalidraw.json` 文件。

### 文件结构

```json
{
  "type": "excalidraw",
  "version": 2,
  "source": "Claude Code",
  "elements": [...],
  "appState": { "theme": "light" }
}
```

### 白板内容设计

- **核心观点**居中大字
- **分论点**作为分支节点
- **数据/案例**作为注释
- **品牌信息**作为标注

### 输出路径

`{选题文件夹}/白板.excalidraw.json`

## 去AI味自检清单（每次必查）

| 检查项 | 问题 | 改法 |
|--------|------|------|
| 平行对仗 | "一个是……另一个是……" | 改成"然后呢……" |
| 精致转折 | "不是X，是Y" | 改成随意口语 |
| 段尾升华 | 每段结尾都有金句 | 删除升华句，自然结束 |
| 预告数量 | "我做了两件事" | 直接说做法，不预告 |
| 全程太顺滑 | 没有碎嘴停顿 | 加"说实话""你试试就知道了" |
| 结尾漏导流口 | 只有"有问必答" | 补回"有需要看号……有问题问大咪，有问必答哦" |

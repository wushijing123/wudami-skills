# 小红书账号深度拆解（开源全域版）

## 核心工作流

```
用户提供主页URL → 启动独立Chrome(9333) → 登录检测 → 深度滚动抓取全量笔记 → Python预处理 → AI语义维度归类 → 11段式报告 → HTML可视化
```

---

## ⚠️ 踩坑防护清单（必读）

| # | 规则 | 违反后果 |
|---|------|----------|
| 1 | Chrome 用独立端口 **9333**，禁用 9222 | 杀死用户默认浏览器 |
| 2 | requests 必须 `trust_env=False` 绕过系统代理 | CDP 连不上 |
| 3 | 滚动 `innerHeight * 3` + 连续5次无增才停 | 笔记只抓一半 |
| 4 | 抓取前检测登录状态（退出码 2 = 未登录） | 崩溃无提示 |
| 5 | 选题维度 AI 语义归类，禁万能标签 | 维度无业务价值 |
| 6 | 环形图从 MD 解析 `**维度名**：N 篇（占比 X%）` | 图表消失 |
| 7 | 必须依赖底层纯文本 DOM 正则暴力提取 `xsec_token` 护照并烙印到外部链接中 | 分享的 HTML 点开必定触发扫码拦截墙 |

## 第一步：环境准备（首次使用）

```bash
# 推荐所有人执行，一键安装所有依赖（Playwright, requests 等）
pip install -r requirements.txt
```

## 第二步：启动独立 Chrome

```bash
python3 scripts/launch_chrome.py
```

- 独立端口 9333 + 独立 profile `~/.xhs-chrome-profile/`
- **绝不 kill 任何进程**，绝不动默认浏览器
- 已在跑 → 秒跳过；未启动 → 自动启动并等待就绪
- 登录态永久保存，重启不丢

---

## 第二步：抓取笔记数据

```bash
python3 scripts/xhs_account_scraper.py \
  --cdp http://localhost:9333 \
  --url "https://www.xiaohongshu.com/user/profile/目标账号ID" \
  --max-notes 999 \
  --output outputs/raw_data.json
```

**自动登录检测**：未登录 → 退出码 2 → 引导用户登录 → 重新执行。

**深度滚动**：每次 3 屏高度 + networkidle 等待 + 连续 5 次无新增才确认到底。

---

## 第三步：统计分析

```bash
python3 scripts/xhs_account_analyzer.py \
  -i outputs/raw_data.json \
  -o outputs/account_report.md \
  -j outputs/account_data.json
```

输出全量笔记标题清单（附点赞数）+ 五维 jieba 词云 + 防幻觉约束。

---

## 第四步：AI 深度分析（11 段式）

读取 `account_report.md`，写出 11 段式拆解报告：

1. **吸睛开场** — 原创钩子，禁止套话
2. **账号速览** — 含"绝对点赞量"声明
3. **账号发展路径** — 基于聚类反推，禁止捏造时间线
4. **爆款因子** — ≥3条：现象→原理→可抄做法
5. **人设打造策略** — 记忆点+信任体系+情绪价值
6. **内容形式分析** — 含 `④ 词云分析` 子节（词云挂载锚点），禁止臆想镜头语言
7. **选题维度拆解** — **核心段落，规则如下**：
   - 基于全量标题语义归纳 4-6 个维度
   - ✅ 维度名具体："茶叶冲泡技巧"、"中小学教辅展示"
   - ❌ 禁止万能标签："流量破圈型"、"垂直干货型"
   - 格式精确：`- **维度名**：N 篇（占比 X%）`（环形图正则依赖此格式）
   - 数量/占比基于实际计数，禁止估算
8. **内容生产 SOP** — 固定骨架+每日变量
9. **变现分析** — 现有路径+升级建议
10. **可复用行动清单** — 5 条可执行
11. **三大启示** — 收尾+行动号召

## 第五步：可视化 + 归档

```bash
python3 scripts/xhs_visualizer.py \
  --md outputs/深度拆解报告.md \
  --json outputs/account_data.json \
  --out outputs/拆解可视化报告.html
```

- **环形图绘制**：从 Markdown 动态解析维度分类数据（不依赖 JSON）。
- **Markdown 链接智能剥离与渲染**：AI 写出的原生 Markdown 笔记外链结构如 `[《标题》](url)` 会被编译器自动剥除方括号外衣，渲染为带有封面配图的高级悬浮预览（Tooltip）交互卡片。普通的 Markdown 超链接同样会被自动转换为小红书品牌红色的 HTML 可跳转锚点文本。

## 第六步：桌面打包交付（极其重要）

为了提供“开箱即用”的小白体验，在上述步骤运行完后，你**必须**在对话框静默执行这条最终指令，把生成的文件打包导出到用户的 Mac 桌面：

```bash
python3 scripts/export_to_desktop.py \
  --md outputs/深度拆解报告.md \
  --html outputs/拆解可视化报告.html \
  --json outputs/account_data.json
```
*(内置特性：自动免配置依赖安装 python-docx，将报告完美转化为 Word 格式文档，并把交互式 HTML 抓取至桌面，以 `日期-账号名-文档类型` 的严格命名规范发送。)*

## 异常处理

| 情况 | 退出码 | 处理 |
|------|--------|------|
| Chrome 未启动 | 1 | 执行 `launch_chrome.sh` |
| CDP 连接失败 | 1 | 检查代理：`NO_PROXY=localhost curl http://localhost:9333/json/version` |
| 未登录 | **2** | 用户在抓取 Chrome 中登录，完成后重新执行 |
| 环形图消失 | — | 检查第七段维度格式 |
| 报告链接点击变扫码墙 | — | 重新跑抓取引擎，确保底层自动扣取 `xsec_token` 填入链接 |

## 防幻觉约束

1. **禁止捏造时间线**：无发布日期，仅靠选题聚类反推
2. **禁止臆想视听语言**：未抓取视频，仅从封面推测
3. **维度 AI 语义归类**：禁万能标签，基于实际标题计数
4. **代理规避**：代码已内置绕过，无需手动干预
5. **浏览器隔离**：端口 9333 + 独立 profile，绝不碰默认 Chrome
